## рҹҢҢ What Is This?

Orbit Chat is a **production-grade real-time messaging platform** built on a dark, cinematic glassmorphism aesthetic вҖ” think deep purples (`#6D28D9`), midnight blacks (`#0F0D1A`), and soft violet glows. Under the hood, it pairs a **React 18 + Zustand** frontend with an **Express + Socket.IO** backend, all backed by **Supabase (PostgreSQL)** for persistence. Every message travels over persistent WebSocket connections with full **sent вҶ’ delivered вҶ’ read** receipt tracking, while a custom **dual-token JWT system** (15-minute access + 24-hour httpOnly refresh cookie) keeps sessions airtight without ever touching localStorage. It ships with a complete **admin dashboard**, group chat support, and a Docker Compose stack that puts Nginx in front of everything вҖ” ready to deploy.

### 3пёҸвғЈ Set Up Supabase Database

Run the full schema in your Supabase SQL Editor:

```sql
-- Enable UUID generation
CREATE EXTENSION IF NOT EXISTS "uuid-ossp";

-- Users table with auth, banning, email verification, and password reset
CREATE TABLE IF NOT EXISTS users (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  full_name TEXT NOT NULL,
  username TEXT UNIQUE NOT NULL,
  email TEXT DEFAULT NULL,
  password TEXT NOT NULL,
  gender TEXT NOT NULL CHECK (gender IN ('male', 'female')),
  profile_pic TEXT DEFAULT '',
  role TEXT DEFAULT 'user' CHECK (role IN ('user', 'admin')),
  is_banned BOOLEAN DEFAULT false,
  is_deleted BOOLEAN DEFAULT false,
  is_email_verified BOOLEAN DEFAULT false,
  email_verification_token TEXT DEFAULT NULL,
  email_verification_expires TIMESTAMP WITH TIME ZONE DEFAULT NULL,
  reset_password_token TEXT DEFAULT NULL,
  reset_password_expires TIMESTAMP WITH TIME ZONE DEFAULT NULL,
  banned_until TIMESTAMP WITH TIME ZONE DEFAULT NULL,
  ban_reason TEXT DEFAULT '',
  created_at TIMESTAMP WITH TIME ZONE DEFAULT now(),
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT now()
);

-- Conversations (DMs + Groups)
CREATE TABLE IF NOT EXISTS conversations (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  is_group BOOLEAN DEFAULT false,
  group_name TEXT,
  group_admin UUID REFERENCES users(id) ON DELETE SET NULL,
  group_avatar TEXT DEFAULT '',
  created_at TIMESTAMP WITH TIME ZONE DEFAULT now(),
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT now()
);

-- Many-to-many bridge for conversation participants
CREATE TABLE IF NOT EXISTS conversation_participants (
  conversation_id UUID REFERENCES conversations(id) ON DELETE CASCADE,
  user_id UUID REFERENCES users(id) ON DELETE CASCADE,
  PRIMARY KEY (conversation_id, user_id)
);

-- Messages with delivery status tracking
CREATE TABLE IF NOT EXISTS messages (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  sender_id UUID REFERENCES users(id) ON DELETE CASCADE,
  receiver_id UUID REFERENCES users(id) ON DELETE CASCADE,
  conversation_id UUID REFERENCES conversations(id) ON DELETE CASCADE,
  message TEXT NOT NULL,
  status TEXT DEFAULT 'sent' CHECK (status IN ('sent', 'delivered', 'read')),
  created_at TIMESTAMP WITH TIME ZONE DEFAULT now(),
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT now()
);

-- Read receipt tracking
CREATE TABLE IF NOT EXISTS message_read_by (
  message_id UUID REFERENCES messages(id) ON DELETE CASCADE,
  user_id UUID REFERENCES users(id) ON DELETE CASCADE,
  PRIMARY KEY (message_id, user_id)
);

-- Comprehensive audit logging
CREATE TABLE IF NOT EXISTS audit_logs (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  user_id UUID REFERENCES users(id) ON DELETE SET NULL,
  action TEXT NOT NULL,
  ip_address TEXT,
  user_agent TEXT,
  details TEXT,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT now()
);
```

| `connection` | `{ auth: { token } }` | Handshake with JWT access token for authentication |

| `disconnect` | вҖ” | Client disconnects; user removed from online map |

### Server вҶ’ Client

## рҹ”’ Security Checklist

- [x] **Helmet.js** вҖ” Sets security HTTP headers (CSP, HSTS, X-Frame-Options, etc.)
- [x] **CORS whitelist** вҖ” Restricted to `CLIENT_URL` origin only
- [x] **httpOnly cookies** вҖ” Refresh tokens never accessible to JavaScript
- [x] **SameSite=Strict** вҖ” CSRF protection on refresh token cookie
- [x] **Secure flag** вҖ” Cookie sent over HTTPS only in production
- [x] **bcrypt hashing** вҖ” Passwords hashed with 10 salt rounds, never stored in plain text
- [x] **express-validator** вҖ” Input sanitization + validation on every endpoint
- [x] **XSS prevention** вҖ” `.escape()` applied to all user-supplied text fields
- [x] **Rate limiting** вҖ” 4 tiers: API (100/15min), Login (5/15min), Signup (3/1hr), AI (10/day)
- [x] **Request size limit** вҖ” `express.json({ limit: "10kb" })` prevents payload bombs
- [x] **JWT expiration** вҖ” Access tokens expire in 15 minutes, refresh in 24 hours
- [x] **IDOR mitigation** вҖ” Group message endpoints verify sender is a participant
- [x] **Admin protection** вҖ” All admin routes gated behind `isAdmin` middleware
- [x] **Ban enforcement** вҖ” Checked at login and before sending messages
- [x] **Timed ban auto-expiry** вҖ” `checkBanned` middleware auto-lifts expired temp bans
- [x] **Soft delete** вҖ” Users marked deleted, not purged, preserving referential integrity
- [x] **Anti-enumeration** вҖ” Forgot-password returns identical response for valid/invalid emails
- [x] **HTTPS enforcement** вҖ” Production server redirects HTTP вҶ’ HTTPS via `X-Forwarded-Proto`
- [x] **Socket authentication** вҖ” JWT verified in `io.use()` middleware before connection
- [x] **Nginx gzip** вҖ” Response compression for all text-based content types
- [x] **Structured logging** вҖ” Winston logs errors with timestamp, IP, and stack traces  
- [x] **Morgan integration** вҖ” HTTP request logs streamed into Winston for unified logging
- [x] **Trust proxy** вҖ” `app.set("trust proxy", 1)` for correct IP detection behind Nginx
