## Data Flow

1. **Authentication:** User signs in/up via the frontend React application. API call (fetch) is sent to backend routes `/api/auth/{login,signup,logout}`.
2. **Session Persistence:** Backend validates and issues an HTTP-only JWT cookie to maintain session.
3. **Real-time Connection:** Once authenticated, frontend establishes a WebSocket connection using socket.io-client back to the Node.js server.
4. **Messaging:** Messages are stored in MongoDB via Mongoose and broadcasted in real-time to active socket clients.
